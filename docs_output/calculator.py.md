# 技术文档：`calculator.py`

## 一、核心功能概述

该文件实现了一个简易的计算器类 `Calculator`，支持基本的加法运算、圆形面积计算以及操作历史记录的管理。通过维护一个内部列表 `history`，可以追溯每次计算的表达式与结果。

## 二、主要类与函数说明

### 2.1 类 `Calculator`

**作用**：提供一个带操作历史记录的计算器实例，封装计算逻辑与状态管理。

**成员变量**：
- `self.history`（`list`）：存储每次计算结果的字符串记录。

---

### 2.2 `__init__(self)`

- **输入**：无
- **输出**：无
- **作用**：初始化 `Calculator` 对象，新建一个空的历史记录列表。

---

### 2.3 `add(self, a, b)`

- **输入**：
  - `a`（`int` / `float`）：加数
  - `b`（`int` / `float`）：加数
- **输出**：`a + b` 的和（`int` / `float`）
- **作用**：执行两个数相加，将表达式与结果存入 `history`，并返回计算结果。

---

### 2.4 `calculate_circle_area(self, radius)`

- **输入**：
  - `radius`（`int` / `float`）：圆的半径
- **输出**：圆的面积（`float`）
- **作用**：使用公式 `π * r²` 计算圆形面积。若 `radius < 0`，抛出 `ValueError` 异常。计算成功后，将记录加入 `history` 并返回面积值。

---

### 2.5 `get_history(self)`

- **输入**：无
- **输出**：`self.history`（`list[str]`）
- **作用**：返回当前计算器的所有历史记录字符串列表。

---

## 三、💡 架构师建议

1. **内存管理优化**  
   当前 `history` 列表会无限增长，长期运行可能耗尽内存。建议增加最大记录数限制（如 `max_history=100`），或在达到上限时自动丢弃最旧的记录。

2. **输入类型校验**  
   `add` 和 `calculate_circle_area` 未对 `a`、`b`、`radius` 进行类型检查。对于非数字类型（如字符串）的输入，Python 可能会抛出意外的异常。建议在方法入口增加类型判断或使用 `numbers.Number` 抽象基类进行校验。

3. **历史记录持久化**  
   若计算器需要跨会话使用，建议将历史记录持久化到文件或数据库中，而非仅保存在内存中。

4. **通用计算接口**  
   当前仅支持加法和圆形面积计算。未来若有更多运算需求，可考虑采用策略模式，将每个运算抽象为独立类，便于扩展与维护。
